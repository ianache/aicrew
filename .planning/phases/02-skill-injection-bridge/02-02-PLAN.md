class BaseTool:

  <behavior>
    **_normalize_schema(schema: dict) -> dict**
    - Test 1: strips `$schema` key from root
    - Test 2: strips `definitions` and `$defs` keys
    - Test 3: strips `examples` from nested property schemas recursively
    - Test 4: injects `additionalProperties: false` when absent
    - Test 5: preserves allowed keys: `type`, `properties`, `required`, `description`, `additionalProperties`, `items`
    - Test 6: passes clean schema through unchanged (idempotent on normalized input)

    **_SkillBaseTool._get_declaration()**
    - Test 7: returns FunctionDeclaration with name, description, and parameters matching normalized schema
    - types.Schema.model_validate succeeds on normalized schema (if this fails, normalization is broken)

    **_SkillBaseTool.run_async — validation path**
    - Test 8: missing required field → returns dict with `missing_fields` containing the field name, does NOT call runner
    - Test 9: extra undeclared field → returns dict with `extra_fields` containing the field name, does NOT call runner
    - Test 10: valid params with additionalProperties: false → passes validation, calls runner with the params

    **_SkillBaseTool.run_async — execution path**
    - Test 11: runner returns ExecutionSuccess → run_async returns `result.data` (the dict)
    - Test 12: runner returns TimeoutError(elapsed_ms=5123) → run_async returns string `"Skill timed out after 5123ms"`
    - Test 13: runner returns ExecutionError(stderr="boom") → run_async returns string `"Skill execution failed: boom"`
    - Test 14: runner returns ValidationFailure(invalid_domain="bad") → run_async returns string `"Skill domain validation failed: bad"`

    **SkillInjector.build_tool()**
    - Test 15: returns tuple of (BaseTool subclass instance, str)
    - Test 16: SKILL.md fetch success → second element is non-empty string content
    - Test 17: SKILL.md fetch 404 / network error → second element is `""`, tool still usable
    - Test 18: skill_def.path `"evaluar_test_case"` constructs URL `"https://raw.githubusercontent.com/ianache/skills-catalog/main/skills/evaluar_test_case/SKILL.md"`

    **Key behaviors (from research pitfalls):**
    - `required` validator errors: extract field name from `e.message` via regex OR compute directly as `[f for f in schema.get("required", []) if f not in params]` (simpler alternative, use this)
    - `additionalProperties` validator errors: compute as `[k for k in params if k not in schema.get("properties", {})]`
    - `ValidationCorrectionRequest` is returned as `.model_dump()` dict — NOT as the Pydantic object — for consistent ADK serialization
    - `build_tool()` is `async def` — test with `await injector.build_tool(skill_def)`
  </behavior>
  <implementation>
    **File: src/skill_injector.py**

    Structure:
    1. Module docstring (design constraints, anti-patterns avoided)
    2. `_ALLOWED_SCHEMA_KEYS` frozenset at module level
    3. `_normalize_schema(schema: dict) -> dict` — recursive, strips disallowed keys, injects `additionalProperties: false`
    4. `async def _fetch_skill_md(path: str, timeout: float = 5.0) -> str` — httpx.AsyncClient, URL pattern `https://raw.githubusercontent.com/ianache/skills-catalog/main/skills/{path}/SKILL.md`, returns `""` on any exception or non-200 status
    5. `class _SkillBaseTool(BaseTool)` — private, NEVER imported by callers
       - `__init__(self, skill_def, runner, normalized_schema)` — calls `super().__init__(name=skill_def.name, description=skill_def.description)`
       - `_get_declaration(self) -> types.FunctionDeclaration` — `types.Schema.model_validate(self._normalized_schema)`
       - `async def run_async(self, *, args: dict, tool_context: ToolContext) -> str | dict`
         - Step 1: Validate with `Draft7Validator`. Compute `missing_fields` and `extra_fields` directly (not from e.path — see pitfall 3)
         - Step 2: If errors → return `ValidationCorrectionRequest(...).model_dump()`
         - Step 3: `result = await self._runner.execute(self._skill_def.path, args, self._skill_def.allow_net_domains)`
         - Step 4: isinstance dispatch → return data or error string
    6. `class SkillInjector`
       - `def __init__(self, runner: DenoRunner) -> None`
       - `async def build_tool(self, skill_def: SkillDefinition) -> tuple[BaseTool, str]`

    **Anti-patterns to avoid (from research):**
    - DO NOT use `FunctionTool` — it drops all args for `**kwargs` closures in ADK 1.33.0
    - DO NOT pass raw `skill_def.input_schema` to `types.Schema.model_validate` — must normalize first
    - DO NOT call `asyncio.run()` inside `build_tool()` — it is already `async def`
    - DO NOT raise exceptions from `run_async` — return strings or dicts
    - DO NOT use `e.path` to extract field names from `required` validator errors — use schema-based computation
