- `name: str` — matches the `name` key in `skill.json`
- `description: str` — plain text description
- `path: str` — bare skill name (e.g., `"evaluar_test_case"`), NOT prefixed with `skills/`. The `skills/` prefix is added only at URL construction time in `skill_injector.py`.

- `allow_net_domains: list[str]` — validated hostnames for `--allow-net` flag; empty list means no network access.

**Model 2: ValidationCorrectionRequest**
Place in the SAME file alongside `SkillDefinition` (per research recommendation — it is a skill-domain concept, not an execution result).
- `missing_fields: list[str]` — fields in `input_schema.required` absent from the LLM payload
- `extra_fields: list[str]` — fields in the LLM payload not present in `input_schema.properties`
- `message: str` — human-readable summary for the LLM to report to the user

Module docstring must explain: "SkillDefinition is the shared contract between CatalogExplorer (Phase 4) and SkillInjector (Phase 2). ValidationCorrectionRequest is returned by SkillInjector._SkillBaseTool.run_async() when JSON Schema validation fails — returned as .model_dump() dict, not as the Pydantic object, for consistent ADK serialization."

Zero imports from ADK, Deno, or execution modules. Only: `from pydantic import BaseModel`.
